## Руководство по запуску авто-тестов

### Условия для запуска авто-тестов:
1. **Установлена Android Studio**
2. **Установлен SDK.** Создана и настроена переменная ANDROID_HOME и указан путь до папки Windows Perfomance Tools.
3. **Установлен Java JDK 11.** Создана и настроена переменная окружения JAVA_HOME и указан путь до папки bin в переменной "path".
4. **В Android Studio  указан путь до SDK Android.**
5. **Эмулятор Android.** Должен быть установлен и настроен эмулятор Android с версией API 29.
6. **Установлен Allure** для генерации отчетов.

### Шаги по настройке проекта.

1. Склонировать проект из репозитория: https://github.com/DmitryLeonov186/DiplomaProject
2. Открыть проект в Android Studio и подождать, пока завершится сборка и синхронизация проекта с Gradle.

### Выполнение тестов.

1. После разворачивания среды на вашем Android Studio перейдите в директорию app/src/androidTest/java/ru.iteco.fmhandroid.ui/test.
2. Кликните на папку test правой кнопкой мыши и выберите  «Run 'Tests in iteco.fmhandroid.ui'», для запуска всех тестов данного пакета. Чтобы выполнить запуск отдельного теста в отдельном классе необходимо выбрать его и нажать на зеленый треугольник напротив теста.

### Генерация отчетов Allure.
1. Перейдите в окно Device Explorer и перейдите в директорию /data/data/ru.iteco.fmhandroid/files/allure-results устройства на котором выполнялись тесты.
2. Нажмите правой кнопкой мыши на папке Files, выберите Save As и укажите путь копирования для отчетов (если выбрать папку allure-results вместо files, то отчеты копируются без папки allure-results "россыпью".
4. Подтвердите выбор.

   Примечание: папку allure-results нужно предварительно почистить от ненужных отчетов (можно просто удалить, а затем после выполнения тестов нажать правой кнопкой на папку files и выбрать Synchronize, чтобы синхронизировать  с актуальными данными. Папка allure-results снова появится и будет наполнена актуальными данными тестов.
