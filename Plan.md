## 2. Типы тестирования, которые необходимо применить

Для данного приложения будут использованы следующие виды тестирования:

**Функциональное тестирование:**
Проверка всех функциональностей (авторизация, навигация между разделами, возможность создавать и редактировать реестровые записи и т.д.). Такая проверка поможет выявить проблемы в работе приложения, которые могут мешать его правильной работе.

**Тестирование пользовательского интерфейса:**
Проверки связанные с отображением интерфейса (Дружелюбность интерфейса, хорошо ли проглядываются текст и изображение , адаптация под разные размеры экранов, и т.д.). Тестирование UI должно показать насколько интерфейс понятен и доступен для Конечного пользователя.

**Тестирование на удобство использования:**
UT поможет выявить насколько пользователю удобно использовать данное приложение и насколько оно отвечает его интересам.

## 3. Используемые интсрументы тестирования

**Android Studio** - среда разработки для Android от IntelliJ, основанная на IDEA, содержаит в себе множество инсnрументов и возможностей.
**SDK (Software Development Kit)** — набор инструментов для разработки программного обеспечения, объединённый в одном пакете.
**JDK 11 (JAVA Development KIT)** - набор инструментов для разработки и запуска программ на языке Java. Предоставляет возможность работы с множестовм библиотек и фреймворков. В данном случае используется 11-я версия.
**Junit 4** - фреймворк для модульного тестирования Java-приложений. Позволяет проводить автоматизированные тесты.
**Espresso** - фреймворк для автоматизированного тестирования пользовательского интерфейса мобильных приложений на платформе Android. Record Espresso Test позволяет записывать действия пользователя и писать код тестов, основываясь на этих данных.
**Layout Inspector** - Позволяет работать с иерархией пользовательсого приложения в реальном времени. Показывает, вложенности друг в друга, их свойства и параметры макета.
**UI Automator** - фреймворк для автоматизации тестирования UI на Android-устройствах. Позволяет создать автоматизированные тесты для приложений.  Полезен для проверки функциональности, а также для автоматизации рутинных задач.
**Allure** - инструмент для создания отчетов тестирования. Позволяет получать анализировать результаты тестов в наглядном виде, экспортировать и импортировать их, оствлеживать статус.

## 5. Оценка рисков тестирования

**1. Нет четких требований к результату тестирования**
#### Границы результаттов расплывчаты и  неясно на что конкретно стоит обратить внимание. Это прямо влияет на конечный результат. Точный объем трудозатрат в тестировании не определить.

**2. Отсутствует ТЗ**
#### Неясно какие возможности реализованы в приложении. Есть вероятность упущения критически важной функциональности.

**3. Нет доступа к БД**
#### Отсутствует возможность протестировать на баги в передачи данных. Функциональность может пострадать.
